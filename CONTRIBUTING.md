# Contributing

Thanks for your interest in contributing to xcel!

## Getting started
1. Fork the repo and create your branch from `main`.
2. Ensure you have Java 17+ installed.
3. Build and test with the Gradle wrapper:
   - Build: `./gradlew build`
   - Test: `./gradlew test`

## Project structure
- `xcel-validation-core` – core validation APIs and rules
- `xcel-validation-poi` – Apache POI-based implementations

## Code style
- Keep changes focused and small when possible.
- Add or update tests for behavior changes.
- Prefer clear, readable code over cleverness.

## Pull requests
- Describe the problem and solution in the PR.
- Link related issues if applicable.
- Include tests and documentation updates when relevant.

## Reporting bugs
Please open a GitHub issue with:
- Steps to reproduce
- Expected vs. actual behavior
- Version/commit
- Logs or stack traces
